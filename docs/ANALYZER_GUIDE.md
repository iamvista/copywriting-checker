# 分析引擎使用指南

## 概述

Vista 文案健檢工具的分析引擎已完全整合 Vista Cheng 的文案寫作方法論，提供 6 大維度的專業分析。

## 六大分析維度

### 1. ⭐ FAB 法則（權重：25%）

**檢測內容：**
- **F (Feature)** - 產品特性、規格、功能（25 分）
- **A (Advantage)** - 競爭優勢、差異化（35 分）
- **B (Benefit)** - 對顧客的具體利益（40 分）⭐ 最重要

**常見錯誤：**
只停留在 F（特性）層面，沒有說明 B（效益）

**改善建議範例：**
- ❌ 錯誤：「搭載 A17 Pro 晶片」（只有 F）
- ✅ 正確：「搭載 A17 Pro 晶片（F），比前一代快 20%（A），讓你工作更有效率，重要時刻不漏接（B）」

**關鍵字庫：**
- Feature: 搭載、配備、採用、具備、擁有...
- Advantage: 獨家、唯一、首創、領先、優於...
- Benefit: 讓你、幫你、節省、提升、輕鬆、解決...

---

### 2. ✨ 標題吸引力（權重：15%）

**7 大元素檢測：**
1. **數字** (20%) - 具體數字提升可信度
   - 範例：「3 個步驟」、「提升 200%」
2. **疑問句** (15%) - 引發思考
   - 範例：「為什麼...？」、「如何...？」
3. **痛點關鍵字** (25%) - 觸動需求
   - 範例：「困擾」、「避免」、「解決」
4. **好奇心觸發** (20%) - 吸引點擊
   - 範例：「秘訣」、「很少人知道」、「原來」
5. **情緒詞彙** (20%) - 增強吸引力
   - 範例：「驚人」、「必學」、「輕鬆」
6. **權威/名人** (15%) - 加分項
   - 範例：「專家推薦」、「台積電思維」
7. **緊迫感** (15%) - 加分項
   - 範例：「限時」、「最後」、「僅剩」

---

### 3. 🎯 消費者洞察（權重：15%）

**4 大檢測項：**
1. **目標受眾** (25%) - 明確定義讀者
2. **痛點情境** (30%) - 描述具體困擾
3. **情緒連結** (25%) - 同理心表達
4. **同理心** (20%) - 「我懂」、「也曾經」

**改善公式：**
```
「身為{role}，你是否常常{pain_point}？」
```

---

### 4. 🚀 行動呼籲（權重：15%）

**6 大元素（基於 Vista 的 CTA 三問）：**

**核心三問：**
1. 我希望目標受眾做哪些事情？
2. 如何確保目標受眾知道自己該做什麼事？
3. 目標受眾為什麼要這樣做？可以得到什麼利益？

**檢測要素：**
1. **明確指示** (40%) - 最重要！
   - 範例：「立即報名」、「馬上下載」
2. **行動動詞** (20%)
   - 範例：「立即」、「現在」、「馬上」
3. **急迫感** (20%)
   - 範例：「限時 24 小時」、「僅剩 10 名」
4. **降低門檻** (20%)
   - 範例：「免費試用」、「30 秒完成」
5. **利益提醒** (15%) - 加分項
   - 範例：「立即報名，獲得價值 5000 元課程」
6. **社會認同** (10%) - 加分項
   - 範例：「已有 10,000 人加入」

---

### 5. 📖 可讀性（權重：15%）

**5 大標準：**
1. **句子長度** (25%)
   - 理想：≤ 25 字
   - 可接受：≤ 35 字
   - 過長：> 35 字
2. **段落結構** (25%)
   - 理想：3-5 行
   - 避免：超過 8 行
3. **標點符號** (20%)
   - 適當使用，驚嘆號 ≤ 3 個
4. **專業術語** (15%)
   - 最少：≤ 2 個術語
   - 過多：> 5 個術語
5. **留白分段** (15%)
   - 使用列表、空行、小標題

---

### 6. 💎 價值主張（權重：15%）

**4 大核心：**
1. **清楚說明好處** (30%)
   - 範例：「讓你在 30 天內掌握技巧」
2. **具體數字/證據** (25%)
   - 數字：「提升 40%」
   - 時間：「3 週見效」
   - 社會證明：「200 位學員見證」
3. **差異化表達** (25%)
   - 範例：「業界唯一」、「獨家專利」
4. **可信度支持** (20%)
   - 客戶見證、認證、媒體報導

**價值主張公式：**
```
[目標受眾] + [痛點] + [解決方案] + [具體成果] + [差異化]
```

## 評分等級

| 分數範圍 | 等級 | 說明 |
|---------|------|------|
| 80-100 | 🏆 優秀 | 文案品質卓越，可直接使用 |
| 60-79 | 👍 良好 | 文案基礎紮實，稍作調整更佳 |
| 40-59 | ⚠️ 待改進 | 有明顯改善空間 |
| 0-39 | 🚨 需重寫 | 建議參考範例重新撰寫 |

## 程式化使用範例

### 基本使用

```typescript
import { analyzeCopywriting } from '@/services/analyzer'

// 分析文案
const result = await analyzeCopywriting('你的文案內容...')

console.log('總分:', result.totalScore)
console.log('等級:', result.grade)
console.log('FAB 得分:', result.dimensions.fab.score)
console.log('FAB 建議:', result.dimensions.fab.suggestions)
```

### 結果結構

```typescript
interface AnalysisResult {
  totalScore: number  // 0-100
  grade: 'excellent' | 'good' | 'needsImprovement' | 'needsRewrite'
  dimensions: {
    fab: DimensionScore
    titleAppeal: DimensionScore
    consumerInsight: DimensionScore
    callToAction: DimensionScore
    readability: DimensionScore
    valueProposition: DimensionScore
  }
  analyzedText: string
  timestamp: Date
}

interface DimensionScore {
  score: number         // 該維度得分
  feedback: string[]    // 評估回饋（✓ 或 ✗）
  suggestions: string[] // 改善建議
}
```

```
knowledge-base/templates/suggestions.json
```

在各 JSON 檔案中修改 `weight` 屬性。

## 最佳實踐

### 1. FAB 法則永遠第一
- 確保每個文案都完整包含 F、A、B
- 特別注意 B（效益）- 這是最常被忽略的

### 2. 標題決定點擊率
- 至少包含 3 個以上吸引力元素
- 數字 + 痛點 + 好奇心 = 黃金組合

### 3. CTA 要明確且有急迫感
- 明確告知下一步（權重 40%）
- 加入時間或數量限制
- 降低行動門檻

### 4. 可讀性影響轉換
- 手機時代：句子要更短
- 善用列表和分段
- 避免專業術語

### 5. 價值主張要具體
- 用數字說話
- 展示差異化
- 提供社會證明

## 常見問題

### Q1: 為什麼 FAB 維度權重最高（25%）？
A: FAB 法則是 Vista Cheng 最核心的文案框架。根據教學內容，「不說 Benefit 是最常見的文案錯誤」，因此給予最高權重。

### Q2: 如何提高 FAB 分數？
A: 確保文案包含：
1. 產品特性（F）- 「搭載...」、「採用...」
2. 競爭優勢（A）- 「獨家...」、「領先...」
3. 顧客利益（B）- 「讓你...」、「幫你...」

### Q3: 建議都是模板化的，如何客製化？
A: 建議模板位於 `knowledge-base/templates/suggestions.json`，可以：
1. 修改模板內容
2. 新增更多建議類型
3. 根據產業調整用詞

### Q4: 如何新增更多關鍵字？
A: 編輯對應的 JSON 規則檔案，在 `keywords` 陣列中新增即可。
