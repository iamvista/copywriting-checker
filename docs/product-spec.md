# 產品功能規格書 (Product Specification)

## 一、產品定位

### 核心問題
企業和學員常寫出缺乏吸引力的文案，但不知道問題出在哪裡，也不知道如何改進。

### 解決方案
提供一個智能文案健檢工具，基於 Vista 的專業框架，給予即時、具體、可執行的改善建議。

### 使用情境
1. **課程學員**：上完課後練習文案，使用工具自我檢驗
2. **企業行銷人員**：撰寫社群貼文、EDM 前先健檢
3. **潛在客戶**：透過免費工具體驗 Vista 的教學方法，產生報名意願

## 二、MVP 功能範圍

### 2.1 核心功能：文案分析

#### 輸入介面
- **文案輸入框**
  - 支援多行文字輸入
  - 即時字數統計
  - 清除按鈕
  - 範例文案按鈕（提供 2-3 個範例）

- **輸入限制**
  - 最少：20 字
  - 最多：1500 字
  - 超過限制顯示友善提示

#### 分析按鈕
- 明確的 CTA：「開始健檢」
- 分析中顯示載入動畫
- 預估時間提示（約 5-10 秒）

### 2.2 分析維度與評分

#### 維度 1：標題吸引力 (Title Appeal Score)
**評分標準** (0-100 分)：
- ✅ 包含數字：+20 分
- ✅ 使用疑問句：+15 分
- ✅ 包含痛點關鍵字：+25 分（如：省時、賺錢、避免、解決）
- ✅ 引發好奇心：+20 分（如：秘密、方法、為什麼）
- ✅ 情緒詞彙：+20 分（如：驚人、必學、獨家）

**回饋範例**：
```
📊 標題吸引力：65/100
✓ 已包含數字（提升可信度）
✗ 缺少疑問句（建議：將標題改為問句形式）
✗ 可加入痛點關鍵字（例：「避免」、「解決」）

💡 改善建議：
「5個方法提升業績」→「為什麼這5個方法能讓業績翻倍？」
```

#### 維度 2：消費者洞察 (Consumer Insight Score)
**評分標準** (0-100 分)：
- ✅ 明確提到痛點：+30 分
- ✅ 使用情緒連結詞彙：+25 分（如：擔心、困擾、渴望）
- ✅ 明確目標受眾：+25 分（如：創業者、主管、家長）
- ✅ 同理心表達：+20 分（如：我懂、也曾經、相信你）

**回饋範例**：
```
📊 消費者洞察：45/100
✓ 有提到目標受眾
✗ 痛點不夠明確（讀者無法產生共鳴）
✗ 缺少情緒連結

💡 改善建議：
加入具體痛點情境，例如：
「身為行銷人員，你是否常常花了3小時寫文案，卻只有3個人按讚？」
```

#### 維度 3：行動呼籲 (Call-to-Action Score)
**評分標準** (0-100 分)：
- ✅ 有明確的下一步指示：+40 分
- ✅ 使用行動動詞：+20 分（如：立即、馬上、現在）
- ✅ 營造急迫感：+20 分（如：限時、最後、倒數）
- ✅ 降低行動門檻：+20 分（如：免費、一鍵、30秒）

**回饋範例**：
```
📊 行動呼籲：60/100
✓ 有明確下一步
✓ 使用行動動詞
✗ 缺少急迫感（讀者可能拖延）
✗ 行動門檻較高

💡 改善建議：
「歡迎報名課程」→「限時優惠！點擊連結，30秒完成報名」
```

#### 維度 4：可讀性 (Readability Score)
**評分標準** (0-100 分)：
- ✅ 平均句長 < 25 字：+25 分
- ✅ 段落長度適中（3-5 行）：+25 分
- ✅ 使用標點符號斷句：+20 分
- ✅ 避免過多專業術語：+15 分
- ✅ 有留白或分段：+15 分

**回饋範例**：
```
📊 可讀性：70/100
✓ 句子長度適中
✓ 有適當分段
✗ 專業術語過多（建議用白話文解釋）

💡 改善建議：
「運用 AIDA 模型建立銷售漏斗」
→「用4步驟吸引顧客：引起注意→產生興趣→激發慾望→促成行動」
```

#### 維度 5：價值主張 (Value Proposition Score)
**評分標準** (0-100 分)：
- ✅ 清楚說明「能得到什麼」：+30 分
- ✅ 有具體數字或證據：+25 分（如：節省 50%、3天見效）
- ✅ 差異化表達：+25 分（為什麼選我們而非競品）
- ✅ 可信度支持：+20 分（如：客戶見證、數據、案例）

**回饋範例**：
```
📊 價值主張：50/100
✓ 有提到好處
✗ 缺少具體數字（難以評估效益）
✗ 未說明差異化（為何要選你）

💡 改善建議：
「課程幫助你成長」
→「200位學員見證：3週內掌握文案技巧，轉換率平均提升40%」
```

### 2.3 綜合評分與報告

#### 總分計算
```
總分 = (標題吸引力 + 消費者洞察 + 行動呼籲 + 可讀性 + 價值主張) / 5
```

#### 等級劃分
- 🏆 **優秀** (80-100 分)：文案品質卓越，可直接使用
- 👍 **良好** (60-79 分)：文案基礎紮實，稍作調整更佳
- ⚠️ **待改進** (40-59 分)：有明顯改善空間
- 🚨 **需重寫** (0-39 分)：建議參考範例重新撰寫

#### 視覺化呈現
- 雷達圖：顯示五大維度的相對強弱
- 進度條：每個維度的得分
- 顏色編碼：
  - 綠色 (80-100)：做得好
  - 黃色 (60-79)：還不錯
  - 橘色 (40-59)：需注意
  - 紅色 (0-39)：需改進

### 2.4 改善建議

#### AI 輔助改寫 (Optional - Phase 2)
- 基於分析結果，提供 2-3 個改寫版本
- 說明每個改寫版本的優化重點
- 使用者可複製使用

#### 學習資源連結
- 針對弱項，推薦相關文章或課程
- 例如：「標題吸引力」低 → 連結到 Vista 的標題撰寫教學文章

## 三、使用者流程 (User Flow)

### 主流程
```
1. 進入首頁
   ↓
2. 閱讀簡短說明（這個工具能幫你什麼）
   ↓
3. 選擇：
   - 輸入自己的文案
   - 點擊「試試範例文案」
   ↓
4. 點擊「開始健檢」
   ↓
5. 等待分析（5-10 秒，顯示載入動畫）
   ↓
6. 查看分析報告
   - 總分與等級
   - 五大維度雷達圖
   - 逐項詳細回饋
   ↓
7. 行動選項：
   - 下載 PDF 報告（Phase 2）
   - 修改文案後重新分析
   - 了解更多 → 導流到 Vista 網站/課程
```

### 替代流程 A：輸入字數不足
```
使用者輸入 < 20 字
   ↓
系統提示：「文案至少需要 20 字才能進行完整分析喔！」
   ↓
使用者補充內容 OR 點擊範例文案
```

### 替代流程 B：API 錯誤
```
分析過程中 API 出錯
   ↓
系統提示：「分析遇到問題，請稍後再試」
   ↓
提供「使用基礎分析」選項（不使用 AI，僅規則式分析）
```

## 四、技術實作重點

### 4.1 分析引擎架構

#### 雙引擎模式
1. **規則引擎** (Rule-based)
   - 快速、成本低
   - 關鍵字比對、字數統計、句型分析
   - 適用：標題吸引力、可讀性

2. **AI 引擎** (Claude API)
   - 深度語意理解
   - 適用：消費者洞察、價值主張
   - 成本控制：使用 Haiku 模型

#### 混合策略
```typescript
async function analyzeCopywriting(text: string) {
  // 平行執行
  const [ruleResult, aiResult] = await Promise.all([
    analyzeByRules(text),      // 規則引擎
    analyzeByAI(text)           // AI 引擎（可選）
  ])

  return mergResults(ruleResult, aiResult)
}
```

### 4.2 效能優化
- 輸入防抖 (debounce)：避免頻繁 API 呼叫
- 結果快取：相同文案 24 小時內不重複分析
- 漸進式載入：先顯示規則分析結果，AI 結果後續補充

### 4.3 錯誤處理
- API 超時：15 秒後 fallback 到純規則分析
- 網路錯誤：友善提示並提供離線版本（僅規則分析）
- 輸入驗證：前端即時驗證，避免無效請求

## 八、風險與應對

| 風險 | 影響 | 應對策略 |
|------|------|----------|
| AI API 成本過高 | 高 | 優先使用規則引擎，AI 為輔助 |
| 分析準確度不足 | 高 | 持續收集回饋，優化規則庫 |
| 使用者不理解評分 | 中 | 加強說明文案，提供範例 |
| 載入時間過長 | 中 | 實作漸進式載入，先顯示部分結果 |
| 使用者流失 | 中 | 優化引導流程，降低操作門檻 |
